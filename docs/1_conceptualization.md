## 2. Business purpose

Starlog는 막연한 꿈을 구체적인 목표로 바꾸고, 그 목표를 꾸준히 이어갈 수 있도록 돕는 웹사이트이다. 사람들은 종종 꿈을 가지라고 말하지만, 실제로는 무엇을, 언제, 어떻게 이루고 싶은지 명확하게 정하지 못한 채 막연한 바람으로만 남겨두는 경우가 많다. 또한 목표를 세웠더라도 일상 속에서 자주 상기하지 않으면 쉽게 중단되며, 처음의 의지마저 잃어버리기 쉽다.

“작심삼일도 10번 하면 한 달이 되고, 그것을 12번 반복하면 1년이 된다”는 생각처럼, 큰 목표는 하루하루의 작은 실천이 쌓여 이루어진다. 따라서 Starlog는 사용자가 자신의 꿈을 더 선명하고 구체적인 목표로 설정할 수 있도록 돕고, 매일의 기록을 통해 그 목표를 지속적으로 상기할 수 있는 환경을 제공하는 것을 목적으로 한다.

Starlog는 단순히 할 일을 체크하는 서비스가 아니라, 매일의 실천을 축적해 나가는 경험에 초점을 맞춘다. 사용자는 하루를 기록하고, 그날의 노력에 따라 별을 부여받으며, 그 별들이 모여 하나의 별자리가 되고 결국에는 자신만의 하늘을 완성하게 된다. 이는 목표 달성 과정을 눈에 보이는 형태로 표현함으로써 성취감과 동기부여를 높이고, 꾸준함의 가치를 자연스럽게 느끼게 한다.

![System Context Diagram](https://github.com/kayeong97/picture_repo/blob/main/software_design/ContextDiagram.BMP)

### Description of terms in the diagram

Starlog System은 하루하루의 기록을 통해 목표를 지속적으로 실천할 수 있도록 지원하는 웹 시스템이다. 사용자는 기록을 쌓아가며 별을 획득하고, 이를 통해 자신의 성장 과정을 시각적으로 확인할 수 있다.

User는 Starlog System의 주 사용자이다. 매일 실천 내역을 기록하며, 완성되어 가는 자신만의 밤하늘을 확인하면서 지속적인 동기 부여를 받는다.

DB는 사용자 정보, 목표 데이터, 일일 기록 등 System에서 생성되는 주요 데이터를 저장하는 외부 저장소이다.

Administrator는 시스템 운영과 관리를 담당하는 외부 주체이다. 사용자 문의, 공지사항 전달 등 원활한 서비스 환경을 유지하는 역할을 수행한다.

Notification Service는 사용자가 목표를 잊지 않고 꾸준히 실천할 수 있도록 알림 및 리마인더를 제공하는 외부 서비스이다.

## 6. Problem statement

### Problems the project should consider

Starlog는 목표를 장기적으로 유지하기 어렵다는 문제를 해결하려는 서비스이지만, 실제 구현 과정에서는 몇 가지 기술적·설계적 과제를 함께 고려해야 한다.

첫째, 사용자의 목표, 일일 기록, 별 보상 내역, 별자리 진행 상태 등 서로 긴밀하게 연결된 데이터를 안정적으로 저장하고 일관되게 관리해야 한다. 둘째, 사용자가 매일 기록하는 흐름이 끊기지 않도록 직관적인 UI/UX를 제공해야 한다. 셋째, 알림 기능은 사용자의 몰입을 돕되 과도한 방해가 되지 않도록 적절한 빈도와 시점을 설계해야 한다. 넷째, 시각적 보상 요소인 별자리 진행 화면은 단순한 장식이 아니라 실제 동기부여 요소로 작동해야 하므로, 사용자의 기록 데이터와 자연스럽게 연결되는 구조가 필요하다.

또한 인증 및 개인정보 보호 역시 중요한 고려 사항이다. 사용자의 계정 정보와 개인 기록은 민감한 데이터가 될 수 있으므로, 로그인 처리, 접근 제어, 비밀번호 보호, 데이터 전송 보안 등을 체계적으로 설계해야 한다.

### Non-Functional Requirements (NFRs)

| NFR Category | Description                                                                                                                                                                                      |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Usability    | 시스템은 사용자가 쉽고 직관적으로 사용할 수 있어야 한다. 목표 설정, 기록 작성, 진행 상황 확인 등의 주요 기능은 복잡하지 않은 화면 구성과 명확한 인터페이스를 통해 제공되어야 한다.               |
| Reliability  | 시스템은 사용자의 목표 정보, 일일 기록, 별 부여 내역 등이 오류 없이 안정적으로 저장되고 조회될 수 있어야 한다. 데이터 손실이나 잘못된 표시가 발생하지 않도록 신뢰성 있는 동작이 보장되어야 한다. |
| Performance  | 시스템은 로그인, 기록 조회, 목표 확인, 마이페이지 접근 등의 기능이 빠르게 수행되어야 한다. 사용자가 서비스를 이용할 때 과도한 지연 없이 원활하게 응답해야 한다.                                  |
| Security     | 시스템은 사용자의 계정 정보와 개인 기록 데이터를 안전하게 보호해야 한다. 로그인 인증, 비밀번호 관리, 데이터 접근 제어 등을 통해 개인정보 유출을 방지해야 한다.                                   |

## 7. Glossary

| Term                 | Description                                                                                                      |
| -------------------- | ---------------------------------------------------------------------------------------------------------------- |
| Starlog              | 사용자가 꿈을 구체적인 목표로 설정하고, 매일의 기록을 통해 목표 달성 과정을 관리할 수 있도록 돕는 웹 기반 시스템 |

| Database             | 사용자 정보, 목표, 일일 기록, 별 부여 내역, 별자리 진행 상태 등을 저장하는 데이터 저장소                         |
